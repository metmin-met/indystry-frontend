<h1 id="nordwerk--order-flow--states-with-stack-coverage">Nordwerk —
Order Flow &amp; States (with Stack Coverage)</h1>
<blockquote>
<p><strong>Purpose:</strong> The complete operation of the demo company
<strong>Nordwerk Manufacturing</strong> (fictional B2B CNC/metal parts
manufacturer, <em>make-to-order</em>) as <strong>a continuous
flow</strong> — from the initial inquiry through <strong>order
receipt</strong> to <strong>completed delivery</strong>, and further to
service and complaints. For each phase, it outlines: which
<strong>states</strong> an order passes through (business language
<strong>+ actual system status</strong>) and <strong>what our stack
does</strong> — including <strong>equipment, production optimization,
and Digital Twin</strong>.</p>
<p>Sister document: <strong><code>PROZESS-ROADMAP.md</code></strong>
(steps P0–P12, personas, coverage). This document provides the
<strong>state/process view</strong> with stack mapping.</p>
<p><strong>Reading aid:</strong> Statuses are shown as <em>business
language</em> <code>technical Odoo status</code>. ✅ = clickable
read-only demo available. All figures/statuses are verified from the
live demo database (<code>met18</code>/<code>nordwerk</code>), not
invented.</p>
</blockquote>

<h2 id="act-ii--order--delivery-core">ACT II · Order → Delivery
(Core)</h2>
<h3 id="p4--production-planning--scheduling">P4 · Production Planning
&amp; Scheduling</h3>
<p><strong>Goal:</strong> Make order production-ready + schedule.
<strong>Steps:</strong> Bill of Materials → Work plan/routing → Material
requirements (MRP) → Scheduling → Machine/shift allocation.</p>
<p><strong>What our stack does here:</strong> <strong>ERP/MRP</strong>
generates requirements + manufacturing orders from the order.
<strong>BI</strong> shows <strong>machine utilization per line</strong>
(early bottlenecks). <strong>Control Center</strong> shows which machine
is free/occupied. <strong>Equipment Twin</strong> provides the
machine/line structure (3 lines, 8 machines). <strong>Personas:</strong>
Thomas (Production Planning/PPC), Frank (Production Manager), Bernd
(Purchasing Trigger).</p>

<h2 id="2--equipment-control-technology--digital-twin-in-depth">2 ·
Equipment, Control Technology &amp; Digital Twin (In-Depth)</h2>
<p>The part that distinguishes Nordwerk from pure ERP — the
<strong>bridge from order to real machine</strong>.</p>
<h3 id="equipment-twin--3d--master--troubleshoot-anlage">Equipment Twin
/ 3D — Master &amp; Troubleshoot (<code>/anlage/</code>)</h3>
<p>Clickable drilldown, read-only, generic (no machine builder
secrets):</p>
<pre><code>Plant → Line (North/South/West) → Machine (8) → Control Cabinet → Component → Wiring Diagram</code></pre>
<p>Per machine, a 3D model with hotspots (control/HMI, spindle, control
cabinet); in the control cabinet <strong>PLC, drives/VSD, terminals,
fuses</strong> → per component a <strong>Mermaid wiring
diagram</strong>. Benefit: Understand setup, quickly navigate from
machine to diagram in case of faults (persona
<strong>Jens</strong>).</p>
<h3 id="control-center--scada--real-time-status-leitstand">Control
Center / SCADA — Real-Time Status (<code>/leitstand/</code>)</h3>
<p>HMI tiles for all machines with <strong>running / maintenance due /
idle / fault</strong> + current temperature/vibration. The same machines
as in the ERP work center and Historian — <strong>one
truth</strong>.</p>
<h3
id="digital-twin--what-if--optimization-twin-demoindystryworks">Digital
Twin — What-if &amp; Optimization
(<code>twin-demo.indystry.works</code>)</h3>
<ul>
<li><strong>before the order (P2):</strong> Simulate
throughput/scrap/costs → realistic quotation.</li>
<li><strong>in operation (P6/P12):</strong> Test parameters and
bottlenecks without stopping the real line.</li>
</ul>
<h3 id="production-optimization--the-closed-control-loop">Production
Optimization — The Closed Control Loop</h3>
<pre class="mermaid"><code>flowchart LR
    M[&quot;Machines\n(Production P6)&quot;] --&gt;|Telemetry| H[(&quot;Historian\nTemp/Vibration&quot;)]
    H --&gt; BI[&quot;BI: Process Drift,\nMass Balance, OEE&quot;]
    H --&gt; PW[&quot;Predictive\nMaintenance → maintenance.request&quot;]
    BI --&gt; TW[&quot;Digital Twin:\nOptimize Parameters&quot;]
    TW --&gt; M
    PW --&gt; M</code></pre>
<ul>
<li><strong>Drift Detection:</strong> BI detects creeping deviation
(process drift) earlier than final inspection.</li>
<li><strong>Mass Balance:</strong> Mass balance dashboard reveals
material/yield losses.</li>
<li><strong>Predictive Maintenance:</strong> Rising
vibration/temperature in Historian → <strong>maintenance
request</strong> (P10) before machine failure (visible as "maintenance
due" in Control Center).</li>
<li><strong>Twin Feedback:</strong> Optimized parameters from simulation
back into production.</li>
</ul>
